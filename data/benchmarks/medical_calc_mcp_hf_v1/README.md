# Medical-Calc-MCP Benchmark

## Dataset Description

Medical-Calc-MCP Benchmark is a project-authored benchmark dataset for structured medical calculator execution. Every case is grounded in a real calculator implementation from the Medical-Calc-MCP repository and is regenerated from validated unit-test style inputs instead of freeform synthetic labels.

This package is designed for:

- tool-use benchmarking for medical calculators
- grounded numeric answer evaluation
- schema-first agent benchmarking
- reproducible clinical-domain coverage audits

Current release summary:

- 738 total validated cases
- 93 unique tools
- 26 primary specialties
- 16 of 16 guideline domains covered
- 616 public-dev cases
- 122 hidden-test cases

## Supported Tasks

Each record asks the model to invoke a specific calculator with structured parameters and produce a numeric result. This makes the dataset suitable for:

- question answering over structured clinical inputs
- tool invocation accuracy evaluation
- regression benchmarking for agentic calculator workflows

## Dataset Creation

Cases are extracted from successful calculator invocations found in repository tests, then re-executed against the live calculator implementations.

Inclusion rules:

- the calculator must map to a registered Medical-Calc-MCP tool
- all parameters must be literal and reproducible
- calculator execution must succeed at generation time
- output must be numeric
- duplicate tool_id plus params combinations are removed

Exclusion rules:

- invalid or exception-oriented test cases
- non-numeric outputs
- unresolved calculator aliases

Additional curated executable seeds are used only when a guideline domain would otherwise be uncovered.

## Split Strategy

Internal benchmark splits:

- train: 506
- validation: 110
- test: 122

Release splits:

- public-dev: train plus validation, 616 cases total
- hidden-test: stricter held-out evaluation set, 122 cases total

The generator rebalances the held-out partition so all 16 guideline domains are present in the hidden evaluation slice.

## Limitations

- This is a calculator-execution benchmark, not a full clinical reasoning benchmark.
- Hidden-test governance is operational rather than cryptographic; repository operators should keep the hidden bundle in a private remote or private branch.
- Labels are numerically grounded, but benchmark quality still depends on correct tool selection and faithful parameter passing.
